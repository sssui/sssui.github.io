http及相关知识

DNS（域名服务器）：是进行域名和与之相应的IP地址转换的服务器。

域名解析

1 在客户端输入一个域名，如www.imooc.com
–> 客户端向本地的DNS服务器发起请求，本地DNS服务器先查看域名与ip的对应关系（查看缓存），如果有该条记录，则将对应的ip返回客户端

2 缓存中没有记录，本地DNS服务器就会像根DNS服务器发起请求，根DNS服务器会返回本地DNS服务器消息：可以向.com域服务器发起请求。 .com域服务器收到请求后将域名的解析器的地址告诉本地DNS服务器，本地DNS服务器收到消息后，像域名解析服务器发起请求，域名解析服务器（这里的 imooc.com )将查询结果(这里的www.imooc.com )返回本地DNS服务器。

我的理解：
简单来说，如果本地DNS服务器没有该域名的缓存，本地DNS服务器会逐次向根DNS服务器，域服务器，域名解析服务器发请求，最后由域名解析服务器将查询结果返回本地DNS服务器。

3 本地DNS服务器收到返回到查询结果时，一是要将结果返回给客户端，二是要更新本地缓存，把记录保存在缓存里。

4 域名解析完成以后拿到域名对应的ip地址，发起http三次握手

关于Query String

语句：
1、querystring.stringify(a,b,c)
作用：序列化

a:url参数（对象)
b:连接符（默认&)
c:键值对符号符号（默认=）

e.g. querystring.stringify({name:’scott’,course:[‘jade’,’node’],from:’’})

结果：’name=scott&course=jade&course=node&from=’

querystring.stringify({name:’scott’,course:[‘jade’,’node’],from:’’},’,’)

结果：’name=scott,course=jade,course=node,from=’

querystring.stringify({name:’scott’,course:[‘jade’,’node’],from:’’},’,’,’:’)

结果：’name:scott,course:jade,course:node,from:’

2、querystring.parse(a,b,c)
作用：反序列化

a:对象字符串
b:需要转换的属性连接符
c:键值对符号

e.g. querystring.parse(‘name=scott,course=jade,course=node,from=’,’,’)

结果：
{ name: ‘scott’, course: [ ‘jade’, ‘node’ ], from: ‘’ }

3、querystring.escape(string，para1,para2)

4、querystring.unescape()

转译与反转译语句
e.g.
querystring.escape(‘<哈哈>’) –>
‘%3C%E5%93%88%E5%93%88%3E’

querystring.unescape(‘%3C%E5%93%88%E5%93%88%3E’) –>
‘<哈哈>’

关于URL网址解析相关知识点

URI : 统一资源标识符–字符串格式规范
URL : 统一资源定位符–网址

语句：
url.parse(urlstr,[parseQueryString],[stashesDenoteHost])–解析URL

url.format(urlobj) –格式拼接，获取正确的能解析的地址 –即生成URL–将url对象格式化为url字符串

url.resolve(from,to) –拼接地址–即拼接URL

url地址的组成部分

protbcol:底层使用的协议 e.g. http
slashes:是否有协议的双斜线
host:服务器的地址或是域名
port:端口
hostname:主机名
hash:哈希值–代表网页的一个位置
search：查询字符串的参数
pathname:路径名
Path:路径
href:超链接

关于node里的http.response.writeHead

还是多写写以免忘记了
关于node里的http.response.writeHead
response.writeHead(statusCode,[reasonPhrase],[headers])

参数
statusCode – HTTP状态码 e.g. 404(未找到),200(请求成功)
reasonPhrase –(可选) 是字符串，用于简单说明 比如：“服务器因为维护暂时无法正常访问….”
headers –(可选) 是个object对象，设置响应头信息 ， 表示响应头的每个属性

res.writeHead(200,{‘Content-Type’:’text/html;charset=UTF8’})–此格式可以识别html，编码格式是utf-8

res.writeHead(200,{‘Content-Type’:’text/plain;charset=UTF8’})–不可识别Html结构

easy node

var http = require(‘http’); //导入require模块
var server = http.createServer(function(req , res){
res.writeHead(200,{‘Content-Type’:’text/plain’});
res.end(‘Hello Sssui\n’);
})

server.listen(1337,’127.0.0.1’);
console.log(‘Server running at http://127.0.0.1:1337/');
