about vue

1、 v-model：数据双向绑定，互相干扰，一方发生变化，另一方也会随之变化。
2、 v-on：事件绑定
3、 v-for：循环功能，循环提取一个数组或一个对象的所有项
Vue的最大的特点是除去了对DOM的直接操作，是对数据进行变化。
v-bind:可以用于响应式的更新html特性

4、

MVP模式（Model,View,Presenter):Model层-数据层，Presenter层-控制层，View-视图层

MVVM模式

5、 Vue.component–创建全局组件
v-bind:绑定父组件中的变量传给props，组件中通过props获取。

–关于条件渲染–
6、 v-if，v-else-if,v-else的标签要连在一起写，中间不要加其他的语句
7、 vue在渲染时会尽量的复用元素，给dom元素加上key值可以保证标签的唯一性 如 ,

,同时选择key值的时候尽量不要用Index的值作为key值

–关于列表渲染–

8、
关于数组的数据变异方法：
方法一：改变数据的引用
方法二：应用以下七种方法，不能直接始用下标来操作数组，否则在页面上显示不了改变的数据。

splice（）–截取或者操作数组的一个方法，如vm.list.splice(1,1,{id=”222” , text=”2”}) –意思是从下标是1的地方开始删除一条，并且往里面增加一条id为222，text为2的数据
push:在数组后面添加一个元素
pop:删除数组最后一个元素
shift：把数组第一项删除
unshift:往数组的第一项加内容
sort：数组排序
reverse：数组取反

方法三:使用vue中的set方法
如 userinfo[1,2,3]
要变更数据将下标为0的数据变为3–
vm.set(userinfo,0,3)，
或使用vue的set实例方法–
vm.$set(userinfo,0,3)

9、如果有偶多个循环的列表同时渲染，可以使用template模板占位符，template标签不会被渲染到页面

10、关于对对象的循环：动态往对象里面添加键名与键值是没有用的，也要修改引用，即将原来的对象数据复制到控制台后再在对象里面添加需要加的键名（key)与键值–[name(键名key):Lee(键值)]。
但是可以直接修改已经存在的键名的键值

方法二：使用set方法：vue.set(obj,key,value)

–总的来说vue的set方法–
对象：
全局使用：Vue.set(obj,key,value)
vue实例：vm.$set(obj,key,value)
数组：
全局使用：Vue.set(arr,index,value)
vue实例：vm.$set(arr,index,value)

row组件中有tr,td

1.1 子组件通过props接收

在Vue的prototype上定义bus（Vue实例）–Vue.prototype.bus = new Vue();

17、

学习微信小程序的一些笔记

最近开始了微信小程序的学习，小的，细碎的知识点太多了…

–image–
image组件默认宽度300px，高度225px；
物理分辨率rpx是逻辑像素px的两倍，即2:1；

rpx能适应不同机型不同分辨率的图片的大小，如果在小程序中使用的是px ，说明在所有的机型上都显示同样的大小，视觉与布局关系会有失调的状况

–text–
只有text包围的文字在手机上才能长按选中，text可以嵌套；
不在标签中的文本在使用中用户可以浏览但是不能选中，

–关于页面布局–

1 在全局里相同的样式可以放到app.wxss文件中
2 window选项是配置导航栏的属性
3 关于高度设计，如果没有设定，则会自己适应内部元素高度宽度。如果需要全部页面都包围，则可以在page标签里面设置。因为page标签中包含了页面里的所有标签

–swiper组件–
可通过改组件实现轮播，不止是图片，还可以放其他的元素
1 swiper-item的宽和高自动设置为100% ，如果要控制高宽，则需要在swiper中设置。
2 vertical=true可设置纵向滚动

http及相关知识

DNS（域名服务器）：是进行域名和与之相应的IP地址转换的服务器。

域名解析

1 在客户端输入一个域名，如www.imooc.com
–> 客户端向本地的DNS服务器发起请求，本地DNS服务器先查看域名与ip的对应关系（查看缓存），如果有该条记录，则将对应的ip返回客户端

2 缓存中没有记录，本地DNS服务器就会像根DNS服务器发起请求，根DNS服务器会返回本地DNS服务器消息：可以向.com域服务器发起请求。 .com域服务器收到请求后将域名的解析器的地址告诉本地DNS服务器，本地DNS服务器收到消息后，像域名解析服务器发起请求，域名解析服务器（这里的 imooc.com )将查询结果(这里的www.imooc.com )返回本地DNS服务器。

我的理解：
简单来说，如果本地DNS服务器没有该域名的缓存，本地DNS服务器会逐次向根DNS服务器，域服务器，域名解析服务器发请求，最后由域名解析服务器将查询结果返回本地DNS服务器。

3 本地DNS服务器收到返回到查询结果时，一是要将结果返回给客户端，二是要更新本地缓存，把记录保存在缓存里。

4 域名解析完成以后拿到域名对应的ip地址，发起http三次握手

5 请求方法：
GET：获取数据
POST:指定资源提交数据
PUT：向指定位置上上传内容
DELETE：请求服务器删除标识的某个资源
HEAD:获取资源（类似GET）

6 状态码：

1xx:请求已接收，继续处理

2xx:请求已接收并处理完 如200：客户端请求成功

3xx:重定向

4xx:客户端错误（请求有语法错误或是无法实现）
如400：客户端请求有语法错误
401：请求未经过授权
403：服务端收到请求但拒绝服务 （权限问题）
404：未找到

5xx:服务器端有错误
如500：服务器端发生不可预计错误 503：服务器端当前还不能处理客户端的请求

关于Query String

语句：
1、querystring.stringify(a,b,c)
作用：序列化

a:url参数（对象)
b:连接符（默认&)
c:键值对符号符号（默认=）

e.g. querystring.stringify({name:’scott’,course:[‘jade’,’node’],from:’’})

结果：’name=scott&course=jade&course=node&from=’

querystring.stringify({name:’scott’,course:[‘jade’,’node’],from:’’},’,’)

结果：’name=scott,course=jade,course=node,from=’

querystring.stringify({name:’scott’,course:[‘jade’,’node’],from:’’},’,’,’:’)

结果：’name:scott,course:jade,course:node,from:’

2、querystring.parse(a,b,c)
作用：反序列化

a:对象字符串
b:需要转换的属性连接符
c:键值对符号

e.g. querystring.parse(‘name=scott,course=jade,course=node,from=’,’,’)

结果：
{ name: ‘scott’, course: [ ‘jade’, ‘node’ ], from: ‘’ }

3、querystring.escape(string，para1,para2)

4、querystring.unescape()

转译与反转译语句
e.g.
querystring.escape(‘<哈哈>’) –>
‘%3C%E5%93%88%E5%93%88%3E’

querystring.unescape(‘%3C%E5%93%88%E5%93%88%3E’) –>
‘<哈哈>’

关于URL网址解析相关知识点

URI : 统一资源标识符–字符串格式规范
URL : 统一资源定位符–网址

语句：
url.parse(urlstr,[parseQueryString],[stashesDenoteHost])–解析URL

url.format(urlobj) –格式拼接，获取正确的能解析的地址 –即生成URL–将url对象格式化为url字符串

url.resolve(from,to) –拼接地址–即拼接URL

url地址的组成部分

protbcol:底层使用的协议 e.g. http
slashes:是否有协议的双斜线

port:端口
hostname:主机名
hash:哈希值–代表网页的一个位置
search：查询字符串的参数
pathname:路径名
Path:路径
href:超链接

关于node里的http.response.writeHead

还是多写写以免忘记了
关于node里的http.response.writeHead
response.writeHead(statusCode,[reasonPhrase],[headers])

参数
statusCode – HTTP状态码 e.g. 404(未找到),200(请求成功)
reasonPhrase –(可选) 是字符串，用于简单说明 比如：“服务器因为维护暂时无法正常访问….”
headers –(可选) 是个object对象，设置响应头信息 ， 表示响应头的每个属性

res.writeHead(200,{‘Content-Type’:’text/html;charset=UTF8’})–此格式可以识别html，编码格式是utf-8

res.writeHead(200,{‘Content-Type’:’text/plain;charset=UTF8’})–不可识别Html结构
